# Source priority by category

## Primary / official
- Product docs and official blogs
- Vendor API references
- Standards bodies (IETF, W3C, ISO)
- Government or university sources

## Reputable secondary
- Major news outlets (Reuters, AP, Bloomberg, Financial Times)
- Respected tech publications (IEEE, ACM, Nature/Science for research news)
- Industry analyst reports (use with bias notes)

## Code / implementation
- Official GitHub repositories
- Maintainer-authored issues and release notes
- Package registry pages (npm, PyPI, Maven, crates.io)

## Community / forums (use with caution)
- StackOverflow, Reddit, Hacker News
- Use to identify leads, then verify with primary sources

## Bias checks
- Vendor blogs may overstate benefits.
- Competitor comparisons may be biased; cross-check.
- Sponsored content should be labeled and treated cautiously.
